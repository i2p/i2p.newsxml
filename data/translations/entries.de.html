I2P 2.12.0 Release

This release of I2P, 2.12.0, continues our work on improving the performance and security of the I2P network. It includes a number of bug fixes and improvements to the I2P software.

Work continues on the implementation of post-quantum cryptography. Improvements to the existing post-quantum implementations improve stability and interoperability with other I2P implementations. We encourage early adopters to continue testing and providing feedback to the I2P team.

A SAMv3 bug which was affecting applications by causing failed lookups has been fixed. Users of SAMv3 applications such as Bitcoin and Bittorrent clients should update to this release to resolve these issues.

As usual, we recommend that you update to this release. The best way to maintain security and help the network is to run the latest release.

I2P 2.3.0 – Sicherheitskorrekturen, anpassbare Sperrlisten, DTG-API

Diese Version enthält Korrekturen für CVE-2023-36325. CVE-2023-36325 ist ein Kontextverwechslungsfehler, der im Bloom-Filter auftrat. Ein Angreifer erstellt eine I2NP-Nachricht, die eine eindeutige Nachrichten-ID enthält, und sendet diese Nachrichten-ID an einen Client. Die Nachricht darf, nachdem sie den Bloom-Filter durchlaufen hat, nicht in einer zweiten Nachricht wiederverwendet werden. Der Angreifer sendet dann dieselbe Nachricht direkt an den Router. Der Router leitet die Nachricht an den Bloom-Filter weiter, wo sie verworfen wird. Dadurch wird die Information preisgegeben, dass die Nachrichten-ID bereits zuvor gesehen wurde, was dem Angreifer einen starken Grund zu der Annahme gibt, dass der Router den Client hostet. Dies wurde behoben, indem die Funktionalität des Bloom-Filters in verschiedene Kontexte aufgeteilt wurde, je nachdem, ob eine Nachricht über einen Client-Tunnel, einen Erkundungstunnel oder direkt an den Router gesendet wurde. Unter normalen Umständen dauert es mehrere Tage, bis dieser Angriff erfolgreich durchgeführt werden kann, und er kann durch verschiedene Faktoren wie den Neustart von Routern während der Angriffsphase und die Anfälligkeit für Fehlalarme vereitelt werden. Benutzern von Java I2P wird empfohlen, sofort ein Update durchzuführen, um den Angriff zu vermeiden.

Im Zuge der Behebung dieses Fehlers, der zu Kontextverwirrungen führte, haben wir einige unserer Strategien zur defensiven Programmierung überarbeitet, um dieser Art von Speicherlecks entgegenzuwirken. Dazu gehören Anpassungen an der netDb, den Mechanismen zur Ratenbegrenzung und dem Verhalten der Floodfill-Router.

In dieser Version wird „not_bob“ als zweiter Standard-Host-Anbieter hinzugefügt, und „notbob.i2p“ sowie „ramble.i2p“ werden auf der Startseite der Konsole angezeigt.

Diese Version enthält außerdem eine anpassbare Sperrliste. Die Sperrung ist semi-permanent; jede gesperrte IP-Adresse bleibt normalerweise so lange gesperrt, bis der Router neu gestartet wird. Benutzer, die bei Sybil-Angriffen ein explosionsartiges Anwachsen der Sperrliste beobachten, können kürzere Zeitlimits aktivieren, indem sie die Sperrliste so konfigurieren, dass Einträge nach einem bestimmten Zeitraum verfallen. Diese Funktion ist standardmäßig deaktiviert und wird derzeit nur für fortgeschrittene Benutzer empfohlen.

Diese Version enthält außerdem eine API, mit der Plugins über die Desktop-Benutzeroberfläche (DTG) angepasst werden können. Es ist nun möglich, Menüpunkte in die Taskleiste einzufügen, wodurch Plugins, die native Anwendungsschnittstellen nutzen, intuitiver gestartet werden können.
