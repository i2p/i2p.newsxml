I2P 2.12.0 Release

This release of I2P, 2.12.0, continues our work on improving the performance and security of the I2P network. It includes a number of bug fixes and improvements to the I2P software.

Work continues on the implementation of post-quantum cryptography. Improvements to the existing post-quantum implementations improve stability and interoperability with other I2P implementations. We encourage early adopters to continue testing and providing feedback to the I2P team.

A SAMv3 bug which was affecting applications by causing failed lookups has been fixed. Users of SAMv3 applications such as Bitcoin and Bittorrent clients should update to this release to resolve these issues.

As usual, we recommend that you update to this release. The best way to maintain security and help the network is to run the latest release.

In deze release wordt verder gewerkt aan het verbeteren van de ondersteuning voor post-kwantumcryptografie. Post-kwantumcryptografie is nu standaard ingeschakeld op het ratchet-niveau.

We hebben de eerste reeks maatregelen genomen om de aanhoudende spamaanvallen tegen te gaan. Er worden momenteel aanvullende maatregelen ontwikkeld.

De SAMv3-API is geüpdatet om ondersteuning te bieden voor de nieuwe datagramtypes 2 en 3.

I2P 2.3.0 - Beveiligingsupdates, aanpasbare blokkeerlijsten, DTG-API

Deze release bevat oplossingen voor CVE-2023-36325. CVE-2023-36325 is een bug die contextverwarring veroorzaakt en die zich voordeed in het bloomfilter. Een aanvaller stelt een I2NP-bericht samen met een unieke messageID en stuurt die messageID naar een client. Het bericht mag, nadat het door het bloomfilter is gegaan, niet opnieuw worden gebruikt in een tweede bericht. De aanvaller stuurt vervolgens hetzelfde bericht rechtstreeks naar de router. De router stuurt het bericht door naar het bloomfilter, waarna het wordt verwijderd. Hierdoor lekt de informatie uit dat de messageID al eerder is gezien, wat de aanvaller een sterke reden geeft om aan te nemen dat de router de client host. Dit is verholpen door de functionaliteit van het bloomfilter op te splitsen in verschillende contexten, afhankelijk van of een bericht via een clienttunnel of een verkennende tunnel is binnengekomen, of rechtstreeks naar de router is verzonden. Onder normale omstandigheden duurt het enkele dagen voordat deze aanval succesvol is uitgevoerd en kan deze worden verstoord door verschillende factoren, zoals het opnieuw opstarten van routers tijdens de aanvalsfase en gevoeligheid voor valse positieven. Gebruikers van Java I2P wordt aangeraden onmiddellijk te updaten om de aanval te voorkomen.

Bij het verhelpen van deze bug met betrekking tot contextverwarring hebben we een aantal van onze strategieën aangepast om defensief te programmeren en dit soort lekken te voorkomen. Dit omvat aanpassingen aan de netDb, de mechanismen voor snelheidsbeperking en het gedrag van floodfill-routers.

In deze release wordt not_bob toegevoegd als tweede standaard hosts-provider, en worden notbob.i2p en ramble.i2p toegevoegd aan de startpagina van de console.

Deze release bevat ook een aanpasbare blokkeerlijst. Het blokkeren van IP-adressen is semi-permanent: elk geblokkeerd IP-adres blijft normaal gesproken geblokkeerd totdat de router opnieuw wordt opgestart. Gebruikers die tijdens Sybil-aanvallen een explosieve groei van de blokkeerlijst waarnemen, kunnen ervoor kiezen om kortere time-outs in te stellen door de blokkeerlijst zo te configureren dat vermeldingen na een bepaalde tijd vervallen. Deze functie is standaard uitgeschakeld en wordt op dit moment alleen aanbevolen voor gevorderde gebruikers.

Deze release bevat ook een API waarmee plug-ins kunnen worden aangepast via de Desktop GUI (DTG). Het is nu mogelijk om menu-items toe te voegen aan het systeemvak, waardoor plug-ins die gebruikmaken van native applicatie-interfaces intuïtiever kunnen worden gestart.
